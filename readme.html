tTalk Add-on

Summary: tTalk

Description: Monitors clipboard content, maintains a history, and announces content in English or Thai.

Author: 'chai chaimee

Version: 2025.5

Minimum NVDA Version: 2022.4

Last Tested NVDA Version: 2025.1

URL: https://github.com/chaichaimee/tTalk

Features

  • Clipboard Monitoring: Keeps track of copied content.
  • Language Toggle: Announce clipboard actions and messages in either English or Thai.
  • Announce Standard Clipboard Actions: Speaks out when you perform copy, paste, cut, undo, select all, and save operations.
  • Smart Copy in Browse Mode: Automatically copies selected text in browse mode (e.g., web browsers) before announcing.

Key Commands

These commands are designed for ease of use:

  • NVDA+Alt+T: Toggles the announcement language between English and Thai.
  • Ctrl+C: Announce "copy" and performs the copy action.
  • Ctrl+V: Announce "paste" and performs the paste action.
  • Ctrl+X: Announce "cut" and performs the cut action.
  • Ctrl+Z: Announce "undo" and performs the undo action.
  • Ctrl+A: Announce "select all" and performs the select all action.
  • Ctrl+S: Announce "save" and performs the save action.

Technical Details (For Developers/Advanced Users)

The add-on uses standard NVDA global plugin architecture. Key functions include:

  • GlobalPlugin: The main class handling global gestures.
  • __init__: Initializes language settings and binds gestures.
  • script_toggleLanguage: Switches the internal language state and announces the change.
  • script_announceCopy: Handles the copy command, with special logic for copying text in NVDA's browse mode before sending the standard Ctrl+C.
  • script_announcePaste: Announces paste and sends the Ctrl+V keypress.
  • script_announceCut: Announces cut and sends the Ctrl+X keypress.
  • script_announceUndo: Announces undo and sends the Ctrl+Z keypress.
  • script_announceSelectAll: Announces select all and sends the Ctrl+A keypress.
  • script_announceSave: Announces save and sends the Ctrl+S keypress.

Troubleshooting

If you encounter issues, please check the NVDA log for messages starting with "tTalk:". Common issues might include:

  • Key binding conflicts: If a gesture fails to bind, an error message will be spoken and logged.
  • Language switching errors: Errors during language announcement will be logged.

For further assistance, please refer to the GitHub repository.
